0. Reading the controls: what every slider means

Before you start dragging things around, here is what each control actually does in the math. Skim once now, refer back as needed.

Random seed

Every "random" number on this page comes from a pseudo-random number generator (PRNG), specifically the Mulberry32 algorithm. A PRNG is deterministic: feed it the same starting integer (the seed) and it produces the same sequence forever. This is a feature, not a bug.

Two reasons we expose the seed. First, reproducibility: a backtest with a fixed seed gives the same result on every machine. If a paper does not specify its seed, you cannot reproduce its plots exactly. Second, sample variation: sliding the seed shows you different realisations from the same underlying probability law. The lesson is that any single sample is one draw from a much wider distribution of possible histories. A 5-year backtest is a single data point about market behaviour, not the truth.

Degrees of freedom (df) of the Student t distribution

The Student t generalises the normal. The mathematical construction: if $Z \sim N(0,1)$ and $V \sim \chi^2_{df}$ are independent, then $T = Z / \sqrt{V/df}$ has the t distribution with df degrees of freedom.

Why does this matter? When df is small, the chi-squared denominator can hit small values often, blowing up the t variable into the tails. As df grows, $V/df$ concentrates around 1 (law of large numbers on $V$), so $T$ approaches $Z$. Concretely: df = 3 has tails so heavy that the kurtosis is infinite. df = 5 has finite kurtosis but it is about 6 (excess kurtosis 3, double the normal). df = ∞ is the normal.

Empirically: equity index returns fit df ≈ 4 to 6 at daily frequency. Crypto fits df ≈ 3. Bonds fit df ≈ 8 to 10. The Gaussian (df = ∞) fits nothing in finance at high frequency.

Years of history

One "year" in our simulator equals 252 trading days. Sliding from 2 to 20 years gives 504 to 5040 daily observations. More observations sharpen statistical estimates by a factor of $\sqrt{n}$. But more history also drags in older regimes that may not match the present (1990s low vol, 2008 GFC, 2020 pandemic, 2022 inflation shock). The bias-variance tradeoff in time series.

GARCH parameters $\alpha$ and $\beta$

The variance equation is $\sigma_t^2 = \omega + \alpha r_{t-1}^2 + \beta \sigma_{t-1}^2$. Three parameters with distinct roles.

$\omega$ anchors long-run variance. The unconditional variance is $\omega / (1 - \alpha - \beta)$ when $\alpha + \beta < 1$.

$\alpha$ is the "ARCH effect", how much yesterday's squared return feeds into today's variance. Bigger $\alpha$ means returns react faster to shocks.

$\beta$ is the "persistence", how much yesterday's variance carries over. Bigger $\beta$ means vol regimes last longer.

For real equity indices, fitted values are typically $\alpha \approx 0.05$ and $\beta \approx 0.93$. The sum 0.98 is what creates the long memory you see in the ACF of $|r_t|$.

Leverage parameter

The Black-Christie asymmetric vol effect. We add an extra $|\text{leverage}| \cdot r_{t-1}^2$ to the GARCH equation only when $r_{t-1} < 0$. Negative leverage means down moves boost vol more than up moves of equal magnitude. SPY has leverage around $-0.4$ in fitted models. VIX has positive leverage (vol of vol rises with VIX). Bonds and gold are roughly symmetric.

Aggregation horizon

How many daily returns we sum to form one observation. Horizon = 1 keeps daily returns. Horizon = 21 ≈ monthly returns. Horizon = 252 ≈ annual. The CLT (Section 9 below) tells us that summing returns drags the distribution toward normal regardless of how heavy-tailed the individual returns are, as long as variance is finite.

ACF lag count

Autocorrelation $\rho(h)$ is computed for each lag $h$. Lag 1 is today vs yesterday. Lag 22 is today vs about a month ago. The slider controls how many lags we display. The dashed grey bands at $\pm 1.96/\sqrt{n}$ are 95% confidence bounds: any bar inside is statistically indistinguishable from zero under the white noise null.

1. What a return actually is

An asset has a price $P_t$ at each time $t$. The return is the percentage change. That is the entire definition. Everything in finance after this is variations on the theme.

The simple return over one period is

$$r_t = \frac{P_t - P_{t-1}}{P_{t-1}} = \frac{P_t}{P_{t-1}} - 1$$

2. Simple returns vs log returns

The log return is

$$r^{\log}_t = \ln\left(\frac{P_t}{P_{t-1}}\right) = \ln(1 + r_t)$$

Two reasons we use both. Simple returns aggregate across assets: a portfolio's simple return is $w_1 r_1 + w_2 r_2$ exactly. Log returns do not aggregate this way. Log returns aggregate across time: the log return over $n$ days equals the sum of daily log returns. Simple returns require multiplication.

Here is the proof of the time-aggregation property:

$$r^{\log}_{[1,n]} = \ln\left(\frac{P_n}{P_0}\right) = \ln\left(\frac{P_n}{P_{n-1}} \cdot \frac{P_{n-1}}{P_{n-2}} \cdots \frac{P_1}{P_0}\right) = \sum_{t=1}^{n} r^{\log}_t$$

3. Annualisation

Returns are reported per period. To compare a daily Sharpe to a monthly Sharpe you have to standardise. The annualisation factor $k$ is the number of periods per year: 252 for daily, 52 for weekly, 12 for monthly.

For the mean: $\mu_\text{ann} = \mu_\text{period} \cdot k$. For the volatility: $\sigma_\text{ann} = \sigma_\text{period} \cdot \sqrt{k}$. The volatility scales with $\sqrt{k}$ because variances add (under independence) but standard deviations do not.

Why $\sqrt{k}$? If daily returns are IID with variance $\sigma^2$, the variance of an $n$-day sum is $n \sigma^2$. The standard deviation is $\sqrt{n} \sigma$. This is the famous "square root of time" rule, and it breaks the moment returns are not IID, which they never are exactly.

4. Stylised fact one: heavy tails

Open any introductory finance textbook and it will assume returns are normally distributed. Real returns are not. The tails are far too heavy.

The Gaussian density falls off like $e^{-x^2/2}$, which means the probability of a 5 sigma event is about 1 in 3.5 million. Empirically, equity markets see 5 sigma days every few years. The Student t distribution with 4 to 6 degrees of freedom fits much better.

4b. Deep dive: degrees of freedom in detail

One thing to nail down before moving on. Lower df means heavier tails. Higher df means thinner tails. The Student t at df $= \infty$ is the normal distribution exactly. df $= 3$ has tails so heavy that the kurtosis is infinite. So when you slide df up from 3 toward 30 in the heavy-tails demo, the empirical histogram becomes better fit by the normal, not worse.

The direction in one line: df is a "Gaussian-ness" dial. Crank df up, you converge to the normal. Crank df down, you diverge from the normal in the tails specifically. df is not a "fatness" dial.

What degrees of freedom actually are

The Student t is constructed from two independent random variables. Let $Z \sim N(0, 1)$ (standard normal) and $V \sim \chi^2_\nu$ (chi-squared with $\nu$ degrees of freedom). Define

$$T = \frac{Z}{\sqrt{V / \nu}}$$

Then $T$ has the Student t distribution with $\nu$ degrees of freedom. The chi-squared $V$ is itself the sum of $\nu$ independent squared normals: $V = G_1^2 + G_2^2 + \cdots + G_\nu^2$.

So df = $\nu$ literally counts how many independent normal squareds you stack into the denominator. This came out of the small-sample statistics literature (Gosset's 1908 paper at Guinness): if you estimate the standard deviation of a sample of size $n$, the t-statistic for the mean has df = $n - 1$. The "degrees of freedom" name comes from the constraint that the deviations from the mean must sum to zero.

Why low df produces heavy tails

The denominator $\sqrt{V / \nu}$ is what creates the tail behaviour. When $\nu$ is small (say 3), the chi-squared denominator can occasionally take very small values (a chi-squared with 3 df has a non-trivial probability of being below 0.5). When the denominator is small, $T$ blows up, which produces the extreme tail values.

As $\nu$ grows, $V / \nu$ concentrates around 1 (law of large numbers: $V$ is the sum of $\nu$ unit-variance things, so $V / \nu \to 1$ almost surely). The denominator approaches 1, and $T$ approaches $Z$, the standard normal.

Quantitatively, the kurtosis of a Student t with $\nu$ df is

$$\text{kurtosis} = 3 + \frac{6}{\nu - 4} \quad \text{for } \nu > 4$$

which gives

  • $\nu = 5$: kurtosis 9, excess kurtosis 6 (twice the normal).
  • $\nu = 6$: kurtosis 6, excess 3.
  • $\nu = 10$: kurtosis 4, excess 1.
  • $\nu = 30$: kurtosis 3.46, excess 0.46. Already nearly normal.
  • $\nu = \infty$: kurtosis 3 exactly.

For $\nu \le 4$ kurtosis is infinite. For $\nu \le 2$ even the variance is infinite. df = 1 is the Cauchy distribution, which has no defined mean.

How to estimate df from data

The standard approach is maximum likelihood. In Python:

from scipy import stats
df_t, mu_t, sigma_t = stats.t.fit(returns)

The fit minimises the negative log-likelihood. Note that maximum likelihood for the t distribution does not have a closed form; scipy uses iterative numerical optimisation. With 1000 daily returns, the standard error on the fitted df is about $\pm 1$ for df near 5, so single-asset df estimates are noisy.

Three rules of thumb when using df in models

1. Round to the nearest integer. The t distribution is defined for any positive real df, but the difference between df = 5 and df = 5.3 is invisible. Use 5.

2. Cap at df = 30. Above 30, the t is indistinguishable from normal in any practical setting. If your fit gives df = 50, just use a normal.

3. Floor at df = 3. Below 3, the variance is infinite (or near infinite), which breaks every variance-based portfolio method. If your data fits df below 3, you are in a regime where Markowitz cannot help and you should switch to CVaR or robust methods (Lessons 12 and 16).

In practice (Macrosynergy art 99): Six different ways to estimate "the same" volatility. Close-to-close, Parkinson (high-low), Garman-Klass (OHLC), Rogers-Satchell, Yang-Zhang, and intraday realised variance. The choice of estimator changes signal-to-noise by a factor of 3 or more. Yang-Zhang is usually the best OHLC-based estimator; intraday realised variance is unbiased but needs clean tick data. Macrosynergy's production default is EWMA close-to-close with an 11-day half-life. That single number flows into the vol-targeting layer (Lesson 21) of every published strategy.

5. Stylised fact two: volatility clustering

Returns themselves look uncorrelated: if today is up, tomorrow is roughly 50/50 up or down. But absolute returns and squared returns are highly autocorrelated. Big days cluster together. Calm days cluster together. This is volatility clustering, the most stable empirical regularity in finance.

The autocorrelation function (ACF) at lag $h$ is

$$\rho_h(x) = \frac{\text{Cov}(x_t, x_{t-h})}{\text{Var}(x_t)}$$

For raw returns $\rho_h \approx 0$ for $h \ge 1$. For absolute returns $\rho_h$ stays high (often above 0.2) for many lags.

Crank $\beta$ to 0.99. The ACF of |returns| stays elevated for 50+ lags. That is a market in persistent vol regime. Drop $\beta$ to 0.6: clustering disappears almost immediately. $\alpha + \beta$ near 1 is what creates the long memory you see in real markets.
In practice (Macrosynergy art 105, 153): The HAR (Heterogeneous Autoregressive, art 153) model uses daily, weekly, and monthly averages of past realised variance to forecast today's variance. It is dead simple (OLS), interpretable, and beats GARCH on most forecasting horseraces. Article 105 finds that LSTM neural networks beat both GARCH and HAR on S&P 500 realised vol in calm and stressed regimes, by roughly 10% out-of-sample R². The production trade-off: HAR is debuggable, LSTM is a black box. Most MS strategies use HAR or EWMA, not LSTM, because debuggability matters more than 10 bps of forecast accuracy.

6. Stylised fact three: the leverage effect

For equities specifically, negative returns predict higher future volatility more than positive returns of the same magnitude do. Black called this the "leverage effect" because if a firm's stock falls, its debt to equity rises, which mechanically raises equity vol. The label has stuck even though the mechanism is debated.

The empirical signature: $\text{Corr}(r_t, \sigma_{t+1}) < 0$, often around $-0.3$ to $-0.5$ for major equity indices.

7. Stylised fact four: aggregational Gaussianity

Daily returns are heavy tailed. Yearly returns are nearly Gaussian. Why? The Central Limit Theorem says that if returns are roughly independent, sums of many of them tend to a normal distribution. Aggregating from daily to yearly means summing about 252 daily returns. That is enough averaging to wash out most tail behaviour.

This is a double edged sword. Long horizon portfolio decisions can use Gaussian assumptions. Short horizon risk management cannot.

8. Stylised fact five: zero linear autocorrelation in returns

Despite all the structure in volatility, raw returns themselves have essentially zero autocorrelation. This is not because markets are inefficient or efficient: it is because if there were strong autocorrelation, traders would arbitrage it away in seconds. Any persistent return autocorrelation has to be tiny, transient, or buried under transaction costs.

This is the empirical foundation for the random walk hypothesis. It does not mean returns are unpredictable. It means linear forecasts based on past returns alone are weak. Almost all alpha comes from non-linear features (regimes, fundamentals, flows) rather than autoregressive structure.

9. Mathematical companion: derivations and proofs

The textbook material that underpins everything you saw. Read once now to fix the foundations. Come back when you build models in later modules.

9.1 Why log returns aggregate across time

The price ratio over $n$ days is a product:

$$\frac{P_n}{P_0} = \prod_{t=1}^{n} \frac{P_t}{P_{t-1}}$$

Take the log of both sides:

$$\ln\frac{P_n}{P_0} = \sum_{t=1}^n \ln\frac{P_t}{P_{t-1}} = \sum_t r_t^{\log}$$

Products become sums. Variance also adds nicely under independence: $\text{Var}\left(\sum_t r_t^{\log}\right) = \sum_t \text{Var}(r_t^{\log})$, which gives the famous $\sigma\sqrt{T}$ scaling. Simple returns do not have this property because $\sum_t r_t \ne (1 + r_1)(1 + r_2)\cdots - 1$. The product expansion creates cross terms.

9.2 Skewness and kurtosis as standardised moments

The third and fourth standardised moments are

$$\text{skew}(X) = \mathbb{E}\left[\left(\tfrac{X - \mu}{\sigma}\right)^3\right], \quad \text{kurt}(X) = \mathbb{E}\left[\left(\tfrac{X - \mu}{\sigma}\right)^4\right]$$

For a normal: skew $= 0$, kurt $= 3$. Throughout finance we report excess kurtosis (kurt $- 3$) so a normal scores zero.

What skewness measures. Geometric asymmetry. Negative skew means the left tail is longer than the right. Equity indices have negative skew because crashes drop further than rallies climb in the same time. Trend following has positive skew. Naked option selling has very negative skew. The sign of skew matters more than the magnitude for portfolio decisions.

The kurtosis myth. Many sources describe high kurtosis as a "peaked distribution". Westfall (2014) shows this is wrong. Kurtosis is almost entirely about tails. The variable $Z^4$ ($Z$ standardised) is small near the centre and explodes far from it. The expectation $\mathbb{E}[Z^4]$ is dominated by tail mass. High kurtosis means rare extreme events, not a tall central peak.

9.3 The Central Limit Theorem, formally

Lindeberg-Lévy CLT: let $X_1, X_2, \ldots$ be IID with finite mean $\mu$ and variance $\sigma^2 > 0$. Then

$$\frac{\sqrt{n}(\bar{X}_n - \mu)}{\sigma} \xrightarrow{d} N(0, 1) \quad \text{as } n \to \infty.$$

This is what produces aggregational Gaussianity. Three caveats matter for finance.

Finite variance. If $X_i$ has infinite variance (e.g. Student t with df $\le 2$), the classical CLT fails. The Generalised CLT replaces the limit with a stable Lévy distribution. Empirically, equity returns appear to have finite variance (df typically above 4) but the convergence to normal is slow.

Independence. Real returns are not independent. They are uncorrelated but the squares are correlated (volatility clustering). The CLT still holds under weak dependence, but the convergence rate is slower than the IID case.

Stationarity. The CLT assumes the process has a fixed distribution. Markets do not. Regimes shift. So aggregational Gaussianity in real data is only approximate, and only inside roughly stationary windows.

9.4 ACF properties and the three flavours of white noise

The autocorrelation function $\rho(h) = \text{Cov}(r_t, r_{t-h}) / \text{Var}(r_t)$ satisfies $\rho(0) = 1$, $|\rho(h)| \le 1$, and $\rho(h) = \rho(-h)$. A process is white noise if $\rho(h) = 0$ for all $h \ne 0$. There are three versions, in order of strength.

  • Uncorrelated white noise. $\rho(h) = 0$ for $h \ne 0$. Says nothing about higher-order dependence.
  • IID white noise. $r_t$ are independent and identically distributed. Implies all order moments are uncorrelated.
  • Gaussian white noise. IID white noise where $r_t \sim N(\mu, \sigma^2)$.

Asset returns are essentially uncorrelated white noise but they are not IID, because $|r_t|$ has strong autocorrelation. This is precisely the volatility clustering stylised fact.

9.5 GARCH(1,1) in formal terms

Engle (1982) introduced ARCH. Bollerslev (1986) extended it to GARCH:

$$r_t = \mu + \sigma_t z_t, \quad z_t \sim \text{IID}(0, 1), \quad \sigma_t^2 = \omega + \alpha r_{t-1}^2 + \beta \sigma_{t-1}^2$$

The unconditional variance is $\omega / (1 - \alpha - \beta)$ when $\alpha + \beta < 1$. Otherwise the variance is unbounded and the process is non-stationary. Empirical fits to equity indices typically give $\alpha + \beta \approx 0.97$ to $0.99$, which means very long memory but technically still stationary. This is called a "near-integrated GARCH" regime.

An equivalent formulation that is sometimes more useful: $\sigma_t^2 = \bar{\sigma}^2 + \alpha(r_{t-1}^2 - \bar{\sigma}^2) + \beta(\sigma_{t-1}^2 - \bar{\sigma}^2)$ where $\bar{\sigma}^2$ is the long-run variance. This says today's variance is yesterday's, partially adjusted toward long-run, plus a kick from yesterday's squared shock.

9.6 Stationarity and ergodicity

A process $\{r_t\}$ is strict stationary if all joint distributions are time-invariant. It is weak (covariance) stationary if mean, variance, and autocovariance do not depend on $t$. Ergodic means time averages converge to ensemble averages.

For empirical estimates to be meaningful, we need both stationarity (so there is a fixed truth to estimate) and ergodicity (so a single sample path tells us about the truth). Real returns are roughly stationary in short windows and clearly non-stationary across regimes. This is the deep reason backtests over 1995 to 2025 are difficult: you are averaging over multiple regimes.

9.7 The QQ plot, formally

A quantile-quantile plot. For each probability $p$ in a grid, plot the empirical $p$-quantile of your data on the vertical axis vs the theoretical $p$-quantile of the reference distribution on the horizontal. If the data follows the reference up to a linear transformation, the points lie on a straight line. Curvature at the ends indicates tail mismatch. Real equity returns produce QQ plots that bend down on the left and up on the right relative to a normal: too many extreme losses and too many extreme gains. The fattest tail is usually the left.

9.8 Why these stylised facts matter for every later lesson

Heavy tails kill Gaussian risk models. Lesson 3 shows you how to handle this with VaR, CVaR, and drawdown. Lesson 13 shows the covariance-estimation consequences. Lesson 16 uses CVaR as the optimisation objective.

Volatility clustering is what makes realised drawdowns much deeper than IID theory predicts. Lesson 21 uses this to motivate volatility targeting.

The leverage effect makes the bond-equity correlation negative in calm markets and positive in stress. Lesson 27 handles this through regime-aware allocation.

Aggregational Gaussianity is the technical reason monthly and quarterly portfolio decisions can use Gaussian-flavoured optimisation (Markowitz, Lessons 5 to 8) while daily decisions cannot.

Exercise A: Variance ratio test

The variance ratio test compares the variance of $k$-day returns to $k$ times the variance of 1-day returns. Under IID, the ratio equals 1. Significant deviation indicates serial dependence.

def variance_ratio(returns, k):
    """Lo-MacKinlay variance ratio at horizon k."""
    one_day_var = returns.var(ddof=1)
    k_day_returns = returns.rolling(k).sum().dropna()
    k_day_var = k_day_returns.var(ddof=1)
    return k_day_var / (k * one_day_var)

Exercise B: Hurst exponent via R/S analysis

The Hurst exponent $H$ measures long-range dependence. $H = 0.5$ is a random walk. $H > 0.5$ is persistent. $H < 0.5$ is mean reverting.

def hurst_exponent(returns, min_lag=10, max_lag=200):
    """R/S Hurst estimate. Returns H in [0, 1]."""
    lags = np.unique(np.logspace(np.log10(min_lag), np.log10(max_lag), 20).astype(int))
    rs = []
    for lag in lags:
        chunks = [returns.iloc[i:i+lag].values for i in range(0, len(returns) - lag, lag)]
        ratios = []
        for c in chunks:
            mean_c = c.mean()
            cumdev = (c - mean_c).cumsum()
            R = cumdev.max() - cumdev.min()
            S = c.std(ddof=1)
            if S > 0:
                ratios.append(R / S)
        rs.append(np.mean(ratios))
    log_lags = np.log(lags)
    log_rs = np.log(rs)
    H, _ = np.polyfit(log_lags, log_rs, 1)
    return float(H)

Exercise C: Realised volatility from intraday data

If you have $m$ intraday observations per day, realised variance is the sum of squared intraday returns. This is unbiased and converges to integrated variance as $m \to \infty$.

def realised_variance(intraday_returns, m_per_day=78):
    """Realised variance from m intraday log-returns per day."""
    n_days = len(intraday_returns) // m_per_day
    rv = np.zeros(n_days)
    for d in range(n_days):
        chunk = intraday_returns[d * m_per_day:(d+1) * m_per_day]
        rv[d] = (chunk ** 2).sum()
    return rv

Exercise D: Implement the leverage effect EGARCH

EGARCH (Nelson 1991) lets the conditional variance respond asymmetrically to negative returns. Implement the simplest version and verify the leverage effect emerges in the simulated path.

def simulate_egarch(n, alpha=0.1, gamma=-0.15, beta=0.95, omega=-0.5, seed=0):
    """log(sigma_t^2) = omega + alpha * |z_{t-1}| + gamma * z_{t-1} + beta * log(sigma_{t-1}^2)."""
    rng = np.random.default_rng(seed)
    z = rng.standard_normal(n)
    log_var = np.zeros(n)
    log_var[0] = omega / (1 - beta)
    ret = np.zeros(n); ret[0] = np.exp(log_var[0] / 2) * z[0]
    for t in range(1, n):
        log_var[t] = omega + alpha * abs(z[t-1]) + gamma * z[t-1] + beta * log_var[t-1]
        ret[t] = np.exp(log_var[t] / 2) * z[t]
    return ret, np.exp(log_var / 2)

Exercise E: Test for Gaussianity

Implement the Jarque-Bera test by hand. Statistic: $JB = (n/6)(\text{skew}^2 + (\text{excess\_kurt})^2 / 4)$. Under the null of normality, $JB \sim \chi^2_2$. Reject at 95% if $JB > 5.99$.

def jarque_bera(returns):
    n = len(returns)
    s = stats.skew(returns)
    k = stats.kurtosis(returns, fisher=True)
    jb = (n / 6) * (s ** 2 + k ** 2 / 4)
    p_value = 1 - stats.chi2.cdf(jb, df=2)
    return dict(jb=jb, p_value=p_value, reject_normal=jb > 5.99)

    'BTC': dict(mu=0.30, vol=0.80, df=3, leverage=-0.2),

def generate_prices(asset_name, n_years=15, seed=None):
    rng = np.random.default_rng(seed if seed is not None else abs(hash(asset_name)) % (2**31))
    p = ASSET_PARAMS[asset_name]
    n = int(n_years * 252)
    eps = rng.standard_t(p['df'], n)
    eps = eps / np.std(eps)
    daily_vol = p['vol'] / np.sqrt(252)
    daily_mu = p['mu'] / 252
    vol = np.zeros(n); vol[0] = daily_vol
    ret = np.zeros(n); ret[0] = daily_mu + vol[0] * eps[0]
    for t in range(1, n):
        shock = vol[t-1] * eps[t-1]
        asym = 1.0 + p['leverage'] * (1 if shock < 0 else 0)
        vol[t] = np.sqrt(1e-7 + 0.05 * (shock * asym)**2 + 0.93 * vol[t-1]**2)
        ret[t] = daily_mu + vol[t] * eps[t]
    prices = 100 * np.exp(np.cumsum(ret))
    idx = pd.bdate_range(end=pd.Timestamp.today().normalize(), periods=n)
    return pd.Series(prices, index=idx, name=asset_name)

prices_dict = {name: generate_prices(name, n_years=15) for name in ASSET_PARAMS}
returns_dict = {name: np.log(p / p.shift(1)).dropna() for name, p in prices_dict.items()}
print('Setup complete. Available: prices_dict, returns_dict for SPY, AGG, GLD, BTC')

def annualised_sharpe(returns, periods_per_year):
    mean = returns.mean() * periods_per_year
    vol = returns.std(ddof=1) * (periods_per_year ** 0.5)
    return mean / vol
