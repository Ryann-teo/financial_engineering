0. Reading the controls

Split scheme

Walk-forward, k-fold, purged k-fold, CPCV. Each gives different out-of-sample Sharpe estimates with different reliability.

Embargo length

Days between train and test windows to prevent label leakage. Should be at least the prediction horizon plus the typical position holding period.

Number of trials $N_{\text{trials}}$

How many strategy variants you tested. Critical input to the deflated Sharpe ratio.

1. Walk-forward backtesting

The honest approach. At each rebalancing date $t$, train on data up to $t - 1$. Predict for date $t$. Roll forward. Standard for time-series problems where ordering matters.

Two flavours:

  • Expanding window. Train on all data up to $t - 1$. More data, slower computation.
  • Rolling window. Train on the last $W$ periods. Adapts to regime, less stable.

Walk-forward gives you one out-of-sample path. The Sharpe of that path is a single noisy estimate, not a distribution.

2. Why standard k-fold leaks

Standard cross-validation (random k-fold) shuffles observations, picks a fold for testing, trains on the rest. For IID data this is fine. For time series it leaks information in three ways:

  1. Observations near the boundary (a test point near a training point in time) share latent state. Train uses information that "leaks" into test.
  2. Multi-period predictions overlap. A model predicting 5-day forward returns at time $t$ has labels using data up to $t + 5$. If $t + 5$ is in the training set, leakage is direct.
  3. Cross-asset exposures. Predicting asset 1 from asset 2 in the same period (where asset 2 is in test) leaks.

The result: standard k-fold gives unrealistically high Sharpe estimates. The model is using the future to predict the past, indirectly.

3. Purged k-fold

Lopez de Prado (2018) fixes leakage with two simple modifications:

  • Purging. When a fold is used as test, remove all training samples whose labels overlap with test samples in time.
  • Embargoing. Add a buffer of $h$ days after the test window before the next training observation begins.

Effective embargo: prediction horizon plus a safety buffer. For weekly forecasts with monthly holding period, embargo around 1 month.

4. Combinatorial Purged CV (CPCV)

Walk-forward gives 1 path. Purged k-fold gives $k$ paths (often correlated). CPCV gives many.

The setup: split the data into $N$ groups. Choose $k$ of them as "test", $N - k$ as "train". Repeat for all $\binom{N}{k}$ combinations. Apply purging and embargoes. Concatenate the $k$ test sub-paths in chronological order to form a "backtest path".

Total paths: $\binom{N}{k} \cdot \binom{N - k}{k - 1} / k$ (combinatorial counting). For $N = 6, k = 2$: 15 paths. For $N = 12, k = 3$: 220 paths. That gives a distribution of Sharpe estimates for one strategy on one dataset.

5. Deflated Sharpe in production

Recap from Lesson 8: $\text{DSR} = \text{PSR}(\mathbb{E}[\max \hat{SR} | \text{null}])$. For $N$ trials and sample size $T$, the bias under the null is approximately $\sqrt{2 \log N / T}$.

Production recipe:

  1. Compute walk-forward Sharpe.
  2. Compute CPCV Sharpe distribution.
  3. Apply Deflated Sharpe with the right $N$ (count every variant tested).
  4. Reject if DSR < 0.95. The bar should be high.
In practice (Macrosynergy art 50, 204): Article 50 ("Evaluating macro signals in three steps") gives the MS protocol: predictive correlation, accuracy, naive PnL. Article 204 ("Systematic trading strategies fooled by live records") quantifies the gap between in-sample and live performance: 30 to 50% Sharpe degradation is typical. CPCV and DSR are the formal tools for setting expectations honestly. MS uses CPCV-style validation in their published research and reports both in-sample and out-of-sample Sharpe.

6. Mathematical companion

6.1 The CPCV path count

For $N$ groups and test size $k$: number of test combinations is $\binom{N}{k}$. Each combination, when arranged chronologically, contributes $k$ contiguous test segments. Total backtest paths formed from concatenating segments: $\binom{N}{k} \cdot k / N$.

6.2 Purging math

Let $h$ be the prediction horizon. A training point $X_t$ overlaps with test point $X_s$ if $|t - s| \le h$. Purge all such training points.

6.3 White's Reality Check (multiple testing)

White (2000) proposes a stationary bootstrap test: under the null that all $N$ strategies have zero Sharpe, what is the probability of seeing the observed best Sharpe? Computed by bootstrapping the data and re-evaluating max Sharpe across strategies. Hansen (2005) refines.

6.4 Family-wise vs false discovery rate

Bonferroni: divide significance level by $N$. Conservative.

Benjamini-Hochberg: control false discovery rate. Accept some false positives but fewer than under no correction.

For trading strategies, FWER (Bonferroni) is appropriate because a single false positive gets capital allocated.

6.5 Bootstrap confidence interval for Sharpe

Block bootstrap to preserve serial dependence. Resample blocks of length $L \approx \sqrt[3]{T}$, recompute Sharpe, take 2.5% and 97.5% quantiles.

7. Coding exercises (extras)

Exercise A: Walk-forward backtest

def walk_forward(strategy, returns, train_window=60, refit_every=12):
    out = []
    for t in range(train_window, len(returns), refit_every):
        train = returns.iloc[t - train_window:t]
        test_end = min(t + refit_every, len(returns))
        model = strategy.fit(train)
        out.extend(strategy.predict(returns.iloc[t:test_end]))
    return out

Exercise B: Purged k-fold

def purged_kfold(n, k=5, embargo=22, label_horizon=22):
    fold_size = n // k
    for i in range(k):
        test_start = i * fold_size
        test_end = test_start + fold_size
        purge_start = max(0, test_start - label_horizon)
        purge_end = min(n, test_end + label_horizon + embargo)
        train_idx = np.concatenate([np.arange(0, purge_start), np.arange(purge_end, n)])
        test_idx = np.arange(test_start, test_end)
        yield train_idx, test_idx

Exercise D: Bootstrap Sharpe CI

def bootstrap_sharpe(returns, n_boot=10000, block_size=22):
    n = len(returns)
    srs = []
    for _ in range(n_boot):
        starts = np.random.randint(0, n - block_size, size=n // block_size)
        sample = np.concatenate([returns[s:s+block_size] for s in starts])
        srs.append(sample.mean() / sample.std() * np.sqrt(252))
    return np.percentile(srs, [2.5, 50, 97.5])

Question 3, numerical

For 1000 trials and $T = 120$ monthly observations, what is the expected max Sharpe under the null?

Answer: Approximately $\sqrt{2 \log 1000 / 120} \approx 0.34$.
