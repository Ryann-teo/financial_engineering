0. Reading the controls

Long-only toggle

Adds the constraint $w_i \ge 0$ for all $i$. Forbids short positions. Common for retail, mutual funds, ETFs. Makes the optimisation strictly less favourable than unconstrained Markowitz, but is often required by mandate.

Maximum position $w_{\max}$

Caps each $w_i \le w_{\max}$. Prevents concentration. Typical: 10% to 40%. Tight caps reduce single-name risk; loose caps allow the optimiser to find any winner-takes-all allocation.

Sector cap

$\sum_{i \in S} w_i \le c_S$ for sector $S$. Mandate-style constraint. Common values: 25 to 35% per sector for diversified strategies.

Leverage cap

$\sum_i |w_i| \le L$. The $L^1$ norm of the weight vector. $L = 1$ is unleveraged. $L = 1.5$ allows a 130/30 long-short structure. $L = 2$ is fully leveraged 100/100.

1. Convex sets and convex functions

A set $C$ is convex if for all $x, y \in C$ and $\theta \in [0, 1]$, we have $\theta x + (1 - \theta) y \in C$. The line segment between any two points stays inside the set. Examples: half-spaces, intersections of half-spaces (polyhedra), Euclidean balls, the simplex.

A function $f$ is convex if $f(\theta x + (1 - \theta) y) \le \theta f(x) + (1 - \theta) f(y)$. Geometrically, the chord lies above the graph. The portfolio variance $w^\top \Sigma w$ is convex in $w$ (because $\Sigma$ is positive semi-definite).

A convex optimisation problem minimises a convex objective over a convex set. The local minimum is the global minimum. Solvers exist that find it in polynomial time. The Markowitz problem is convex. Most practical portfolio optimisations in this course are convex.

2. Lagrangians and duality

For a constrained problem $\min_x f(x)$ subject to $g_i(x) \le 0$ and $h_j(x) = 0$, the Lagrangian is

$$\mathcal{L}(x, \lambda, \nu) = f(x) + \sum_i \lambda_i g_i(x) + \sum_j \nu_j h_j(x)$$

$\lambda_i \ge 0$ are Lagrange multipliers for inequality constraints. $\nu_j$ are multipliers for equality constraints (no sign restriction).

The dual function is $g(\lambda, \nu) = \min_x \mathcal{L}(x, \lambda, \nu)$. Always concave. The dual problem is $\max_{\lambda \ge 0, \nu} g(\lambda, \nu)$. Weak duality says the dual optimum is at most the primal optimum. Strong duality says they are equal, and holds for convex problems satisfying Slater's condition (a strictly feasible point exists).

3. The KKT conditions

For a convex problem, $x^\star$ is optimal iff there exist $\lambda^\star, \nu^\star$ such that:

$$\nabla f(x^\star) + \sum_i \lambda_i^\star \nabla g_i(x^\star) + \sum_j \nu_j^\star \nabla h_j(x^\star) = 0 \quad \text{(stationarity)}$$ $$g_i(x^\star) \le 0, \quad h_j(x^\star) = 0 \quad \text{(primal feasibility)}$$ $$\lambda_i^\star \ge 0 \quad \text{(dual feasibility)}$$ $$\lambda_i^\star g_i(x^\star) = 0 \quad \text{(complementary slackness)}$$

The fourth condition is the most important for intuition. Complementary slackness says either the constraint is tight ($g_i(x^\star) = 0$) or the multiplier is zero ($\lambda_i^\star = 0$). A constraint that is not tight contributes nothing to the optimum and has zero shadow price.

The set of constraints with $\lambda_i^\star > 0$ is called the active set. Identifying the active set is the central computational task in interior-point and active-set solvers.

5. Reading dual values

The Lagrange multiplier $\lambda_i^\star$ on constraint $i$ is the shadow price: by how much would the optimal objective improve if you relaxed the constraint by one unit? In portfolio terms, if your sector cap binds at 30% with $\lambda^\star = 0.005$, then loosening the cap by 1% would add 5 basis points of utility.

This is how risk teams price the cost of mandate restrictions. Run the unconstrained optimisation, then the constrained one. The difference in objective, attributed to each constraint via its dual multiplier, is the cost of compliance.

In practice (Macrosynergy): MS strategies are typically built signal-by-signal with simple constraints (vol target, signal cap at $\pm 2$ SD, sigmoid sizing). They almost never run a full constrained QP because the input vector $\mu$ is too noisy to justify the complexity. The rare exceptions: when implementing for a regulated entity with mandate constraints (sector caps, ESG tilts, country limits), they wrap the existing signal weights with a tracking-error-minimising overlay. Lesson 25 covers this pattern.

5b. Deep dive: reading dual values in production

The Lagrange multiplier on each constraint is not just a mathematical artifact. It has a precise economic meaning: shadow price, the marginal value of relaxing the constraint by one unit.

Worked example: a 50-stock long-only fund

Constraints: long only ($w_i \ge 0$), max position 5%, sector cap 25% (10 sectors), beta neutral. The optimiser returns weights and a vector of dual values, one per constraint. Read them.

  • Long-only multipliers ($\lambda_i^{LO}$): non-zero for assets where the unconstrained optimum is short. Each $\lambda_i^{LO}$ tells you how much utility you would gain per unit of additional shortability of asset $i$. If $\lambda_i^{LO} = 0.005$, allowing 5% short on that name would add 2.5 bp of utility.
  • Max position multipliers ($\lambda_i^{\max}$): non-zero where the cap binds. If $\lambda_i^{\max} = 0.012$, allowing 1% more allocation to that name would add 12 bp of utility. These are your "cant get enough of this" names.
  • Sector multipliers: non-zero where sector caps bind. Tells you which sectors are over-attractive vs the cap.
  • Beta-neutrality multiplier: usually small but non-zero. Quantifies the cost of staying market-neutral.

Aggregate constraint cost

Sum of dual values times constraint slack = total utility loss from constraints. This is the "price of compliance" for a regulated mandate. Risk teams report this in basis points to their PMs and clients.

Sensitivity analysis

For each binding constraint, ask: would the PM benefit from relaxing it? If the dual value is large (high shadow price) and the constraint is "soft" (e.g. an internal risk limit rather than a regulatory rule), it might be worth raising the limit. If the dual value is small, the constraint costs nothing and can be tightened for additional safety margin.

The "constraint binding map" diagnostic

In production, after every optimisation, log the active constraints and their dual values. Trends over time tell you whether your alpha is getting boxed in by the same constraints repeatedly (a sign you should rethink the mandate) or whether constraints are rotating (which is healthy).

6. Mathematical companion

6.1 Why long-only constraints make the problem strictly harder

Without $w \ge 0$, the closed-form Markowitz solution is one matrix-vector product. With $w \ge 0$, the active set is unknown a priori, and the problem becomes a quadratic program with no closed form. Solvers iterate to identify the active set. At each interior-point iteration, the cost is $O(N^3)$ for an $N$-asset universe.

6.2 The dual of mean-variance

For Markowitz with budget constraint, the dual problem maximises $- (1/2 \lambda) (\mu - \kappa \mathbf{1})^\top \Sigma^{-1} (\mu - \kappa \mathbf{1}) + \kappa$ over the scalar $\kappa$. The optimal $\kappa^\star$ has economic meaning: it is the "risk premium" required to make the budget binding.

6.3 Slater's condition for portfolios

Strong duality holds if there exists a strictly feasible point. For most portfolio problems this is trivial: the equal-weight portfolio $\mathbf{1}/N$ is strictly inside any reasonable feasible set. Strong duality lets us move freely between primal (weights) and dual (shadow prices) views.

6.4 KKT for variance minimisation under a return target

Minimise $w^\top \Sigma w$ subject to $w^\top \mu = \mu_p$ and $\mathbf{1}^\top w = 1$. KKT gives

$$2 \Sigma w - \nu_1 \mu - \nu_2 \mathbf{1} = 0$$

so $w = (1/2) \Sigma^{-1} (\nu_1 \mu + \nu_2 \mathbf{1})$. The multipliers $\nu_1, \nu_2$ are pinned by the two equality constraints. Solving the resulting 2x2 linear system gives the closed form for the efficient frontier (see Lesson 5 math companion 6.1).

6.5 Conic programming preview

Tracking-error budgets ($w^\top \Sigma w \le \text{TE}^2$) make the problem a second-order cone program (SOCP). CVaR optimisation (Lesson 16) is a linear program (LP). HRP and risk-parity (Lessons 14 to 15) are convex but not always quadratic. The general label "convex optimisation" covers all of these. Modern solvers (MOSEK, ECOS, SCS) handle the generic case.

7. Coding exercises (extras)

Exercise A: Constrained Markowitz with cvxpy

import cvxpy as cp

def constrained_mv(mu, Sigma, lam, long_only=True, max_pos=0.4, leverage=1.5):
    n = len(mu)
    w = cp.Variable(n)
    obj = cp.Maximize(mu @ w - 0.5 * lam * cp.quad_form(w, Sigma))
    cons = [cp.sum(w) == 1, w <= max_pos]
    if long_only:
        cons.append(w >= 0)
    else:
        cons.append(cp.norm(w, 1) <= leverage)
    prob = cp.Problem(obj, cons)
    prob.solve()
    return w.value, [c.dual_value for c in cons]
