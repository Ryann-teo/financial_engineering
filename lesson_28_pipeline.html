1. The seven-layer architecture

Every systematic strategy can be decomposed into seven layers, each with a distinct contract. Get one wrong and the whole stack fails.

Layer 1: Data

Raw inputs: prices, fundamentals, macro indicators, vintages. Point-in-time. Survivorship-cleaned.

Layer 2: Signal

Transformations: returns, z-scores, ranks, smoothing. The macro themes per asset class.

Layer 3: Forecast

Combine signals into expected returns. Equal weight, IC weight, ML aggregation.

Layer 4: Optimisation

Convert forecasts into target weights. Markowitz, ERC, HRP, sigmoid sizing.

Layer 5: Risk

Vol target, drawdown control, factor exposure caps, mandate constraints.

Layer 6: Execution

Rebalancing rule, order generation, slippage modelling, FX hedging.

Layer 7: Monitoring

Live diagnostics: signal IC, ex-ante vol, factor drift, drawdown alarm.

2. Data layer

Garbage in, garbage out. Three rules:

  1. Point-in-time vintages. Macro releases get revised. Use the data as it was known at time $t$, not as revised today.
  2. Survivorship cleaning. Include delisted assets in historical universes. Otherwise you back-test on winners.
  3. Time-zone alignment. A "daily" return for a US asset is from 16:00 ET to 16:00 ET. For Asia, different. Misalign and you get hindsight bias.

3. Signal layer

Each signal is a time-asset matrix $S_{ti}$. Standard transforms:

  • Z-score within date and within universe panel.
  • Cap at $\pm 2$ to $\pm 3$ SD.
  • Smooth via EWMA if signal is noisy.
  • Convert to rank if you want robustness to outliers.

The signal layer outputs a clean numerical signal in the range $\pm 1$ or $\pm 2$. No information about position size yet.

4. Forecast layer

Take signals, output expected returns. Three ways:

  • Equal weight composite. Macrosynergy default.
  • Regression-based. Train signal-to-return mapping with ridge or lasso (Lesson 10).
  • ML. Random forest, gradient boosting, neural net.

Output: $\hat \mu_t$, the expected return vector for period $t+1$.

5. Optimisation layer

Convert $\hat \mu_t$ into target weights $w_t^*$. Choose:

  • Markowitz mean-variance (with shrinkage, BL, or robust).
  • Risk parity / ERC (no $\hat \mu$ used).
  • HRP (no inversion of $\Sigma$).
  • Sigmoid sizing (signal directly to position).

Macrosynergy default: sigmoid sizing on the signal, no MV optimisation. Conceptual parity within and across themes.

6. Risk layer

Apply on top of optimisation:

  1. Vol target the strategy at, say, 10% (Lesson 21).
  2. Cap leverage at $L_{\max}$.
  3. Drawdown control: scale down if rolling DD > threshold (Lesson 16).
  4. Factor exposure caps if mandated (Lesson 25).

The risk layer is the difference between a backtest and a deployed system. Most blowups happen here.

7. Execution layer

  • Rebalance frequency: daily, weekly, monthly. Match signal half-life.
  • No-trade band (Lesson 20). Don't trade tiny weight changes.
  • Cost model: linear + square-root impact.
  • FX hedging: convert all PnL to base currency.

8. Monitoring layer

Live diagnostics that fire alerts:

  • Rolling 6 to 12 month signal IC vs historical.
  • Realised vs target vol.
  • Factor exposure drift vs targets.
  • Top 10 PnL contributors and detractors.
  • Drawdown alarm at DD > 15%.

Monitoring is where strategies die quietly: you can't fix problems you can't see.

In practice (Macrosynergy art 39): Article 39 ("How to build a macro trading strategy with open-source python") is the canonical MS pipeline reference. It documents the seven-layer stack with code. The MS open-source `macrosynergy` Python package implements layers 1 to 4. Layers 5 to 7 are typically built per-product. The whole flagship art 3 strategy is constructed using this exact architecture.

9. Coding exercises (extras)

Exercise A: Pipeline class skeleton

from dataclasses import dataclass
import pandas as pd

@dataclass
class Config:
    universe: list[str]
    rebalance: str = 'monthly'
    target_vol: float = 0.10
    max_lev: float = 3.0
    cost_bp: float = 10
    dd_threshold: float = 0.10

class Pipeline:
    def __init__(self, config: Config, data: dict):
        self.config = config
        self.data = data
    def signal(self) -> pd.DataFrame: ...
    def forecast(self) -> pd.DataFrame: ...
    def optimise(self) -> pd.DataFrame: ...
    def risk(self, w) -> pd.DataFrame: ...
    def execute(self, w) -> pd.Series: ...
    def monitor(self, returns) -> dict: ...
    def run(self):
        s = self.signal()
        f = self.forecast()
        w = self.optimise()
        w = self.risk(w)
        ret = self.execute(w)
        diag = self.monitor(ret)
        return ret, diag

Exercise B: Implement each layer end-to-end

Build a pipeline for a single signal (e.g. trend) on 5 assets. Implement each layer minimally. Verify layers can be swapped.

Exercise C: Add a new signal without breaking the pipeline

Extend the pipeline to combine 3 signals. The forecast layer should accept any number of signals.

Exercise D: Build the monitoring dashboard

Compute rolling IC, realised vol vs target, drawdown. Plot live.

Exercise E: Failure mode injection

Simulate a "bad data" scenario (e.g. 50% of returns dropped). The pipeline should fail gracefully and the monitor should flag it.

10. Self test

Question 2, coding
